# 数据分析和报告查看模块重构 - 产品需求文档

## 概述
- **摘要**: 重构数据分析和报告查看两个模块的前后端代码，实现类似智能问数的对话窗口进行数据分析，以及基于分析结果和对话记录生成专业报告的功能。
- **目的**: 提供更智能、更专业的数据分析和报告生成能力，满足用户对数据分析和报告的需求。
- **目标用户**: 企业数据分析师、业务决策者、数据驱动的管理者。

## 目标
- 实现类似智能问数的对话窗口，支持用户输入分析需求和选择数据源
- 系统自动执行Python代码进行统计分析并返回结果
- 支持上传报告模板，基于智能问数和数据分析结果生成专业报告
- 以markdown形式展示报告，支持人为修改和一键下载
- 实现存储模块，保存用户的智能问数和数据分析结果
- 确保后端模块定义清晰，可被前端准确调用

## 非目标（范围外）
- 不实现复杂的可视化图表生成功能
- 不支持实时数据更新和监控
- 不涉及权限管理和多用户协作的复杂场景
- 不实现高级的自然语言处理功能

## 背景与上下文
- 现有系统已经具备基本的数据分析和报告生成功能
- 后端使用FastAPI框架，前端使用Vue.js
- 系统已集成大语言模型（LangChain）
- 已有数据库模型和CRUD操作的基础实现

## 功能需求
- **FR-1**: 数据分析模块
  - 前端提供对话窗口，用户输入分析需求
  - 支持选择数据源内的数据
  - 系统自动生成并执行Python代码进行统计分析
  - 返回分析结果，包括数据摘要和可视化建议
  - 存储分析结果到数据库

- **FR-2**: 报告查看模块
  - 支持上传报告模板
  - 选择智能问数模块的问答对话结果
  - 选择数据分析模块的分析结果
  - 系统调用AI模型生成专业分析报告
  - 以markdown形式展示报告
  - 支持人为修改报告内容
  - 提供一键下载功能

- **FR-3**: 存储模块
  - 存储用户的智能问数结果
  - 存储用户的数据分析结果
  - 提供查询和管理接口
  - 支持结果的标签和分类

## 非功能需求
- **NFR-1**: 性能要求
  - 数据分析任务执行时间不超过30秒
  - 报告生成时间不超过10秒
  - 前端响应时间不超过1秒

- **NFR-2**: 安全要求
  - 确保用户只能访问自己的分析结果和报告
  - 防止Python代码执行的安全风险
  - 保护用户数据的隐私

- **NFR-3**: 可扩展性
  - 支持添加新的分析算法和报告模板
  - 支持集成不同的大语言模型
  - 支持扩展数据源类型

## 约束
- **技术约束**:
  - 使用现有的FastAPI后端框架
  - 使用现有的Vue.js前端框架
  - 使用现有的数据库模型和CRUD操作
  - 依赖现有的大语言模型集成

- **业务约束**:
  - 保持与现有系统的兼容性
  - 确保重构不影响其他功能模块

- **依赖**:
  - 依赖现有的Python代码执行服务
  - 依赖现有的大语言模型服务
  - 依赖现有的数据库服务

## 假设
- 系统已经具备基本的用户认证和授权功能
- 系统已经配置了可用的大语言模型
- 系统已经连接了数据源
- 用户具备基本的数据分析知识

## 验收标准

### AC-1: 数据分析功能
- **给定**: 用户在前端打开数据分析对话窗口
- **当**: 用户输入分析需求并选择数据源
- **然后**: 系统自动生成并执行Python代码，返回分析结果
- **验证**: `programmatic`
- **说明**: 分析结果应包含数据统计信息和相关洞察

### AC-2: 分析结果存储
- **给定**: 系统完成数据分析
- **当**: 分析完成后
- **然后**: 系统将分析结果存储到数据库
- **验证**: `programmatic`
- **说明**: 存储的结果应包含分析参数、执行代码、结果数据等信息

### AC-3: 报告模板上传
- **给定**: 用户打开报告查看模块
- **当**: 用户上传报告模板
- **然后**: 系统保存模板并在生成报告时使用
- **验证**: `programmatic`
- **说明**: 模板应支持markdown格式

### AC-4: 报告生成
- **给定**: 用户选择分析结果和对话记录
- **当**: 用户点击生成报告按钮
- **然后**: 系统调用大语言模型生成专业报告
- **验证**: `programmatic`
- **说明**: 报告应包含分析结果的摘要和专业解读

### AC-5: 报告展示和编辑
- **给定**: 报告生成完成
- **当**: 用户查看报告
- **然后**: 报告以markdown形式展示，支持编辑和下载
- **验证**: `human-judgment`
- **说明**: 编辑功能应允许用户修改报告内容，下载功能应支持多种格式

### AC-6: 存储模块功能
- **给定**: 用户执行智能问数或数据分析
- **当**: 操作完成后
- **然后**: 系统存储结果，用户可在报告生成时选择
- **验证**: `programmatic`
- **说明**: 存储的结果应可通过API查询和管理
