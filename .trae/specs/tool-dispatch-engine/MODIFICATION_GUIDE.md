# 工具调度型决策引擎平台 - 修改指南

## 概述
本文档详细说明了将SQLBot改造为工具调度型决策引擎平台的所有修改内容、修改位置、修改方式以及修改原因。

## 项目整体架构分析

### 原有架构
- **前端**: Vue 3 + TypeScript + Element Plus + Vite
- **后端**: FastAPI + Python + PostgreSQL
- **通信方式**: RESTful API，使用axios进行前后端通信
- **布局**: 左侧导航菜单 + 右侧内容区域

### 新架构
- 保持原有技术栈不变
- 新增工具调度平台布局：顶部导航 + 左侧工具列表 + 右侧执行面板
- SQLBot原有功能作为平台的工具模块

## 主要修改内容

### 1. 新增文件

#### 1.1 布局组件
- **文件路径**: `/frontend/src/components/layout/ToolDispatchLayout.vue`
- **修改方式**: 新建文件
- **修改原因**: 创建新的工具调度平台布局组件
- **功能说明**:
  - 顶部导航栏：显示Logo、应用名称、用户信息
  - 左侧工具列表：按分类显示工具，支持展开/收起
  - 右侧执行面板：显示选中工具的界面
  - 支持工具切换和路由导航

#### 1.2 工具包装组件
- **文件路径**: `/frontend/src/views/tools/ChatTool.vue`
- **修改方式**: 新建文件
- **修改原因**: 包装原有的智能问数功能，使其能够在新平台中使用
- **功能说明**: 直接复用 `/frontend/src/views/chat/index.vue` 组件

- **文件路径**: `/frontend/src/views/tools/DashboardTool.vue`
- **修改方式**: 新建文件
- **修改原因**: 包装原有的仪表盘功能
- **功能说明**: 直接复用 `/frontend/src/views/dashboard/index.vue` 组件

- **文件路径**: `/frontend/src/views/tools/DatasourceTool.vue`
- **修改方式**: 新建文件
- **修改原因**: 包装原有的数据源管理功能
- **功能说明**: 直接复用 `/frontend/src/views/ds/index.vue` 组件

### 2. 修改文件

#### 2.1 路由配置
- **文件路径**: `/frontend/src/router/index.ts`
- **修改方式**: 
  1. 导入新的布局组件和工具包装组件
  2. 添加新的路由配置 `/tool` 路径
- **修改原因**: 
  - 需要为新的工具调度平台配置路由
  - 保持原有的路由配置不变，确保向后兼容
- **修改详情**:
  - 新增路由 `/tool`，默认重定向到 `/tool/chat`
  - 支持三个工具路由：
    - `/tool/chat`: 智能问数
    - `/tool/dashboard`: 仪表盘
    - `/tool/datasource`: 数据源管理

## 关键类与函数说明

### ToolDispatchLayout.vue
- **activeTool**: 当前选中的工具ID
- **toolCategories**: 工具分类和工具列表配置
- **selectTool()**: 选择工具并导航到对应路由
- **handleCollapseChange()**: 切换左侧工具列表的展开/收起状态

### 工具包装组件
- 所有工具包装组件都采用相同的模式：直接复用原有的功能组件
- 这种方式的优点：
  - 保持原有功能完整性
  - 无需重复开发
  - 维护成本低
  - 向后兼容性好

## 前后端通信方式

### 原有通信方式（保持不变）
- **通信协议**: RESTful API
- **API前缀**: `/api/v1`
- **HTTP客户端**: axios
- **认证方式**: Token认证
- **后端入口**: `/backend/main.py`
- **API路由定义**: `/backend/apps/api.py`

### 无需修改的内容
- 所有后端API保持不变
- 所有前端API调用保持不变
- 数据库结构保持不变
- 认证机制保持不变

- 启动后，访问 `http://localhost:端口/#/tool/chat` 即可使用新的工具调度平台
- 原有的访问路径仍然有效，确保向后兼容

## 使用说明

### 访问工具调度平台
1. 启动前端和后端服务
2. 在浏览器中访问 `/#/tool/chat`
3. 左侧会显示工具列表
4. 点击不同的工具可以在右侧面板中查看和使用对应的功能

### 工具列表
当前平台包含以下工具：
- **智能问数**: 原SQLBot的核心功能，支持自然语言查询数据
- **仪表盘**: 数据可视化和报表展示
- **数据源**: 数据源管理和配置

### 切换工具
- 在左侧工具列表中点击任意工具
- 右侧面板会立即显示该工具的界面
- 工具状态会保持（如果工具支持）

## 后续扩展建议

### 添加新工具
1. 在 `/frontend/src/views/tools/` 目录下创建新的工具包装组件
2. 在 `ToolDispatchLayout.vue` 的 `toolCategories` 中添加新工具配置
3. 在路由配置中添加新工具的路由

### 工具管理功能
当前实现已经包含了基本的工具调度功能，如果需要更完善的工具管理，可以考虑：
- 添加工具的添加、编辑、删除功能
- 实现工具权限管理
- 支持工具的启用/禁用
- 添加工具使用统计

### 报告查看功能
当前未实现报告查看功能，可以通过以下方式添加：
1. 找到原有的报告查看组件
2. 创建对应的工具包装组件
3. 在工具列表和路由中添加配置

## 注意事项

1. **向后兼容性**: 所有原有的路由和功能都保持不变，用户可以继续使用原有界面
2. **功能复用**: 所有工具都直接复用原有的组件，确保功能一致性
3. **样式隔离**: 新的布局组件有独立的样式，不会影响原有界面
4. **路由设计**: 新平台使用 `/tool` 前缀的路由，与原有路由完全隔离
