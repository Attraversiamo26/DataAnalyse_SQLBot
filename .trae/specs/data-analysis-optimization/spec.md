# 数据分析模块全流程优化 - 产品需求文档

## 1. 概述
- **项目目标**：全面梳理和优化数据分析模块的全流程机制，确保三种分析需求生成方式都能准确传入用户需求，并通过大模型实现精准意图识别，执行统计分析和可视化，最终在前端展示结果。
- **项目范围**：涵盖数据分析的完整流程，包括大模型集成、函数定义、识别、调用、执行、前后端通信和结果展示。
- **目标用户**：数据分析师、业务用户、开发人员。

## 2. 核心功能需求

### 2.1 分析需求生成方式
- **FR-1.1**：分析类型和列选择生成分析需求
- **FR-1.2**：分析需求建议生成分析需求
- **FR-1.3**：用户直接输入分析需求

### 2.2 大模型集成
- **FR-2.1**：精准意图识别，识别分析类型和需要分析的列
- **FR-2.2**：分析需求建议生成，基于字段名称和类型
- **FR-2.3**：分析代码生成，用于复杂分析场景

### 2.3 数据分析功能
- **FR-3.1**：描述性统计分析
- **FR-3.2**：相关性分析
- **FR-3.3**：分布分析
- **FR-3.4**：趋势分析
- **FR-3.5**：机器学习预测分析
- **FR-3.6**：分类分析
- **FR-3.7**：异常检测
- **FR-3.8**：时间序列分析
- **FR-3.9**：聚类分析
- **FR-3.10**：回归分析

### 2.4 可视化功能
- **FR-4.1**：直方图生成
- **FR-4.2**：柱状图生成
- **FR-4.3**：折线图生成
- **FR-4.4**：散点图生成
- **FR-4.5**：热力图生成
- **FR-4.6**：箱线图生成

### 2.5 前后端通信
- **FR-5.1**：数据基本统计信息获取
- **FR-5.2**：分析类型列表获取
- **FR-5.3**：分析需求生成
- **FR-5.4**：数据分析执行
- **FR-5.5**：分析结果获取
- **FR-5.6**：分析报告生成

## 3. 非功能需求

### 3.1 性能需求
- **NFR-1.1**：数据分析执行时间不超过30秒
- **NFR-1.2**：数据加载时间不超过10秒
- **NFR-1.3**：前端响应时间不超过2秒

### 3.2 可靠性需求
- **NFR-2.1**：系统稳定性，无崩溃或异常退出
- **NFR-2.2**：错误处理机制完善，提供清晰的错误信息
- **NFR-2.3**：数据处理异常时能够优雅降级

### 3.3 可维护性需求
- **NFR-3.1**：代码结构清晰，模块划分合理
- **NFR-3.2**：日志记录完整，便于问题定位
- **NFR-3.3**：文档完整，便于后续维护和扩展

### 3.4 扩展性需求
- **NFR-4.1**：分析类型可扩展
- **NFR-4.2**：大模型可替换
- **NFR-4.3**：可视化类型可扩展

## 4. 技术实现要点

### 4.1 技术栈
- **前端**：Vue 3 + TypeScript + Element Plus
- **后端**：FastAPI + Pandas + NumPy + Scikit-learn + Matplotlib
- **大模型**：OpenAI API 或其他兼容模型
- **数据存储**：PostgreSQL

### 4.2 核心流程
1. **数据获取**：从数据源或直接输入获取数据
2. **意图识别**：通过大模型识别分析意图和需要分析的列
3. **分析执行**：调用相应的分析函数执行分析
4. **可视化生成**：生成相应的图表
5. **结果返回**：将分析结果和图表返回给前端
6. **报告生成**：基于分析结果生成分析报告

### 4.3 关键技术点
- **函数调用机制**：类似Function Calling的方式，提取参数并调用相应的分析函数
- **数据类型处理**：自动类型推断和转换
- **图表生成**：将图表转换为base64编码的图片字符串
- **大模型集成**：优化提示词，提高意图识别准确率
- **错误处理**：完善的错误处理和日志记录
