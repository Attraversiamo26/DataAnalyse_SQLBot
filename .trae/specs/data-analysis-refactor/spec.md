# 数据分析工具分析相关功能重构 - 产品需求文档

## 1. 概览

本项目旨在重构SQLBot系统中的数据分析工具分析相关功能，重点参考DataAgent-main文件下的有关数据分析的统计分析与机器学习预测部分代码。通过优化前后端架构，提供更直观的用户界面和更强大的分析能力，帮助用户从数据中获取有价值的洞察。

- **主要功能**：数据基本统计、智能分析建议、统计分析、机器学习预测、图表可视化
- **典型应用场景**：业务数据分析、报表生成、数据洞察发现、辅助决策支持
- **技术特点**：结合大语言模型的推理能力与Python的数据分析能力，实现智能化的数据分析流程

## 2. 目标

- **核心目标 1**：重构数据分析工具，优化数据基本统计功能，包括行数、列数、前五条数据dataframe展示等
- **核心目标 2**：增强分析需求建议功能，提供常用的统计分析选项以及列名的选项
- **核心目标 3**：实现智能列选择提示，根据不同分析类型要求用户选取一列或多列
- **核心目标 4**：优化数据分析结果的图表可视化展示，支持多种图表类型
- **核心目标 5**：参考DataAgent-main的实现，增强统计分析和机器学习预测能力

## 3. 非目标（范围外）

- 不修改现有的数据源连接和管理功能
- 不改变现有的大模型配置和管理方式
- 不引入新的外部依赖库
- 不修改报告生成和管理功能

## 4. 背景与上下文

当前系统已经具备基础的数据分析功能，但存在以下问题：

1. 数据分析功能需要参考DataAgent-main的实现进行优化
2. 分析需求建议功能需要进一步增强
3. 列选择提示不够智能，没有根据分析类型动态调整
4. 图表可视化功能需要优化，支持更多图表类型

通过本次重构，将解决这些问题，提供更加集成和高效的数据分析解决方案。

## 5. 功能需求

### 5.1 数据基本统计功能

**FR-1**：数据基本统计分析
- 支持用户选择数据源和表名
- 计算数据的基本统计信息：行数、列数
- 展示前5条数据的dataframe
- 提供列信息：列名、数据类型、非空值数量、空值数量、空值百分比
- 计算数据质量指标：空值总数、空值百分比、重复行数、重复行百分比

### 5.2 分析需求建议功能

**FR-2**：分析类型选择
- 提供常用的统计分析选项：描述性统计、相关性分析、分布分析、趋势分析、机器学习预测
- 每种分析类型显示详细描述
- 明确每种分析类型需要的列数（1列或多列）

**FR-3**：智能列选择提示
- 根据选择的分析类型，动态调整列选择器的行为
- 对于需要单列的分析类型，限制只能选择一列
- 对于需要多列的分析类型，提示用户选择至少两列
- 根据列的数据类型，过滤不适合的列选项

**FR-4**：分析需求生成
- 根据用户选择的分析类型和列，自动生成分析需求描述
- 生成的需求描述清晰明确，包含分析类型和涉及的列

### 5.3 数据分析功能

**FR-5**：统计分析执行
- 支持描述性统计：计算均值、中位数、标准差、最小值、最大值等
- 支持相关性分析：计算相关系数，生成相关性矩阵
- 支持分布分析：分析数据分布情况，生成直方图和分位数统计
- 支持趋势分析：分析数据随时间的变化趋势
- 支持机器学习预测：使用机器学习模型对数据进行预测

**FR-6**：图表可视化
- 支持多种图表类型：柱状图、折线图、散点图、饼图、热力图
- 根据分析类型自动选择合适的图表类型
- 支持用户手动切换图表类型
- 图表展示清晰美观，包含适当的标题和标签

## 6. 非功能需求

**NFR-1**：性能要求
- 数据基本统计分析响应时间不超过 3 秒
- 数据分析执行时间不超过 30 秒
- 页面响应时间不超过 2 秒

**NFR-2**：用户体验
- 界面简洁直观，操作流程清晰
- 提供及时的操作反馈和错误提示
- 支持响应式设计，适配不同屏幕尺寸

**NFR-3**：可扩展性
- 模块化设计，便于添加新的分析类型
- 支持不同类型的数据源
- 支持不同类型的图表可视化

## 7. 约束

**技术约束**：
- 后端使用 Python 和 FastAPI
- 前端使用 Vue 3 和 Element Plus
- 数据库使用 PostgreSQL
- 数据分析使用 pandas 库
- 机器学习使用 scikit-learn 库
- 图表可视化使用 AntV G2

**业务约束**：
- 保持与现有系统的兼容性
- 遵循现有的用户权限管理
- 符合公司的安全规范

**依赖**：
- 依赖现有的大模型服务
- 依赖现有的数据源管理功能
- 依赖现有的用户认证系统

## 8. 假设

- 用户具备基本的数据分析知识
- 系统已经配置了可用的大模型
- 系统已经连接了至少一个数据源
- 服务器具备足够的资源执行数据分析

## 9. 验收标准

### 9.1 数据基本统计功能验收

**AC-1**：数据基本统计分析
- **Given**：用户选择数据源和表名
- **When**：用户点击"获取数据统计"按钮
- **Then**：系统显示数据的行数、列数、前5条数据、列信息和数据质量指标
- **Verification**：`human-judgment`

### 9.2 分析需求建议功能验收

**AC-2**：分析类型选择
- **Given**：用户进入数据分析页面
- **When**：用户查看分析类型列表
- **Then**：系统显示所有支持的分析类型及其描述
- **Verification**：`human-judgment`

**AC-3**：智能列选择提示
- **Given**：用户选择分析类型
- **When**：用户选择列
- **Then**：系统根据分析类型限制列选择数量，并提供相应提示
- **Verification**：`human-judgment`

**AC-4**：分析需求生成
- **Given**：用户选择分析类型和列
- **When**：用户点击"生成分析需求"按钮
- **Then**：系统生成清晰的分析需求描述
- **Verification**：`human-judgment`

### 9.3 数据分析功能验收

**AC-5**：统计分析执行
- **Given**：用户生成分析需求
- **When**：用户点击"开始分析"按钮
- **Then**：系统执行分析并返回结果
- **Verification**：`programmatic`

**AC-6**：图表可视化
- **Given**：分析执行完成
- **When**：系统显示分析结果
- **Then**：系统以图表形式展示分析结果，支持切换图表类型
- **Verification**：`human-judgment`
