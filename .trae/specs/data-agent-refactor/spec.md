# 数据分析与报告生成模块重构 - 产品需求文档

## 1. 概览

本项目旨在重构和增强 SQLBot 系统中的数据分析和报告生成功能，使其成为一个更加集成、高效的数据分析解决方案。通过优化前后端架构，提供更直观的用户界面和更强大的分析能力，帮助用户从数据中获取有价值的洞察。

- **主要功能**：智能数据分析、报告生成与管理、结果存储与复用
- **典型应用场景**：业务数据分析、报表生成、数据洞察发现、辅助决策支持
- **技术特点**：结合大语言模型的推理能力与 Python 的数据分析能力，实现智能化的数据分析流程

## 2. 目标

- **核心目标 1**：重构数据分析模块，提供类似智能问数的对话式界面，支持用户输入分析需求并选择数据源
- **核心目标 2**：增强报告生成模块，支持上传报告模板，选择历史分析结果生成专业报告
- **核心目标 3**：实现分析结果存储功能，方便用户在报告生成时选择和复用历史分析结果
- **核心目标 4**：确保前后端架构清晰，模块划分明确，易于维护和扩展

## 3. 非目标（范围外）

- 不修改现有的智能问数核心功能
- 不涉及数据源连接和管理功能的修改
- 不改变现有的大模型配置和管理方式
- 不引入新的外部依赖库

## 4. 背景与上下文

当前系统已经具备基础的数据分析和报告生成功能，但存在以下问题：

1. 数据分析功能缺乏对话式界面，用户体验不够直观
2. 报告生成功能不支持模板上传和历史结果选择
3. 分析结果没有持久化存储，无法在后续报告中复用
4. 前后端模块划分不够清晰，调用关系复杂

通过本次重构，将解决这些问题，提供更加集成和高效的数据分析解决方案。

## 5. 功能需求

### 5.1 数据分析模块

**FR-1**：对话式数据分析界面
- 提供类似智能问数的对话窗口
- 支持用户输入自然语言分析需求
- 支持选择数据源和数据表
- 实时显示分析进度和结果

**FR-2**：智能 Python 代码生成与执行
- 根据用户需求自动生成分析代码
- 在安全环境中执行 Python 代码
- 支持常见的数据分析库（pandas、numpy 等）
- 捕获和处理执行错误

**FR-3**：分析结果展示
- 支持表格形式展示分析结果
- 支持图表形式展示分析结果
- 支持导出分析结果

**FR-4**：分析结果存储
- 自动存储分析结果到系统
- 支持给分析结果添加标签和描述
- 支持查看历史分析结果

### 5.2 报告生成模块

**FR-5**：报告模板管理
- 支持上传报告模板
- 支持预览和编辑模板
- 支持删除和管理模板

**FR-6**：报告生成功能
- 支持选择智能问数的对话结果
- 支持选择数据分析的分析结果
- 调用大模型生成专业报告
- 以 Markdown 形式展示报告

**FR-7**：报告编辑与导出
- 支持手动编辑生成的报告
- 支持导出为多种格式（PDF、Word 等）
- 支持分享报告链接

## 6. 非功能需求

**NFR-1**：性能要求
- 数据分析执行时间不超过 30 秒
- 报告生成时间不超过 60 秒
- 页面响应时间不超过 2 秒

**NFR-2**：安全性
- Python 代码执行环境隔离
- 防止恶意代码执行
- 数据传输加密

**NFR-3**：可用性
- 系统可用性达到 99.9%
- 支持错误处理和用户提示
- 提供详细的操作指南

**NFR-4**：可扩展性
- 模块化设计，便于添加新功能
- 支持不同类型的数据源
- 支持不同类型的分析方法

## 7. 约束

**技术约束**：
- 后端使用 Python 和 FastAPI
- 前端使用 Vue 3 和 Element Plus
- 数据库使用 PostgreSQL
- 大模型集成使用现有框架

**业务约束**：
- 保持与现有系统的兼容性
- 遵循现有的用户权限管理
- 符合公司的安全规范

**依赖**：
- 依赖现有的大模型服务
- 依赖现有的数据源管理功能
- 依赖现有的用户认证系统

## 8. 假设

- 用户具备基本的数据分析知识
- 系统已经配置了可用的大模型
- 系统已经连接了至少一个数据源
- 服务器具备足够的资源执行数据分析

## 9. 验收标准

### 9.1 数据分析模块验收

**AC-1**：对话式数据分析界面
- **Given**：用户打开数据分析页面
- **When**：用户输入分析需求并选择数据源
- **Then**：系统显示分析结果和图表
- **Verification**：`human-judgment`

**AC-2**：Python 代码执行
- **Given**：用户提交分析请求
- **When**：系统生成并执行 Python 代码
- **Then**：系统返回正确的分析结果
- **Verification**：`programmatic`

**AC-3**：分析结果存储
- **Given**：分析完成
- **When**：用户查看历史分析结果
- **Then**：系统显示之前的分析结果
- **Verification**：`programmatic`

### 9.2 报告生成模块验收

**AC-4**：报告模板上传
- **Given**：用户进入报告模板管理页面
- **When**：用户上传报告模板
- **Then**：系统成功保存模板
- **Verification**：`programmatic`

**AC-5**：报告生成
- **Given**：用户选择历史分析结果和模板
- **When**：用户点击生成报告
- **Then**：系统生成并展示 Markdown 格式的报告
- **Verification**：`human-judgment`

**AC-6**：报告编辑与导出
- **Given**：报告生成完成
- **When**：用户编辑并导出报告
- **Then**：系统保存编辑并提供下载链接
- **Verification**：`programmatic`
