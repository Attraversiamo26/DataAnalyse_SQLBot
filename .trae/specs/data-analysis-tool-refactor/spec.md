# 数据分析工具分析相关功能代码重构 - 产品需求文档

## 1. 概览

本项目旨在重构 SQLBot 系统中的数据分析工具，重点参考 DataAgent-main 文件下的有关数据分析的统计分析与机器学习预测部分代码，提升数据分析功能的用户体验和分析能力。

- **主要功能**：数据基本统计分析、智能分析建议、多维度数据分析、图表可视化
- **典型应用场景**：业务数据分析、数据洞察发现、统计分析、机器学习预测
- **技术特点**：结合 DataAgent 的统计分析与机器学习预测能力，提供直观的数据分析界面

## 2. 目标

- **核心目标 1**：重构数据分析工具，在用户选择数据源和表名后，自动生成数据的基本统计信息
- **核心目标 2**：提供智能分析需求建议，包括常用的统计分析选项和列名选择
- **核心目标 3**：支持不同分析场景的列选择需求，明确提示用户需要选择一列或多列
- **核心目标 4**：根据用户选择生成分析需求，执行数据分析并以图表形式返回结果

## 3. 非目标（范围外）

- 不修改现有的数据源连接和管理功能
- 不改变现有的大模型配置和管理方式
- 不引入新的外部依赖库
- 不涉及报告生成和存储功能的修改

## 4. 背景与上下文

当前系统已经具备基础的数据分析功能，但存在以下问题：

1. 缺乏数据基本统计分析功能，用户无法快速了解数据概况
2. 分析需求建议不够智能，用户需要手动输入完整的分析需求
3. 列选择逻辑不够清晰，用户不知道不同分析需要选择多少列
4. 分析结果展示形式单一，缺乏直观的图表可视化

通过本次重构，将解决这些问题，提供更加智能和直观的数据分析工具。

## 5. 功能需求

### 5.1 数据基本统计分析

**FR-1**：数据基本统计信息
- 在用户选择数据源和表名后，自动计算并展示数据的基本统计信息
- 包括：行数、列数、各列的数据类型、前五条数据的DataFrame展示
- 支持数据预览和基本质量检查

**FR-2**：数据质量分析
- 分析数据中的缺失值、重复值等质量问题
- 提供数据质量报告和建议

### 5.2 智能分析建议

**FR-3**：常用统计分析选项
- 提供常用的统计分析选项，如：
  - 描述性统计（均值、中位数、标准差等）
  - 相关性分析
  - 分布分析
  - 趋势分析
  - 机器学习预测

**FR-4**：列选择建议
- 根据选择的分析类型，智能提示用户需要选择的列数（一列或多列）
- 提供列名的下拉选择功能
- 支持多列选择的场景

### 5.3 数据分析执行

**FR-5**：分析需求生成
- 根据用户选择的分析类型和列名，自动生成数据分析需求
- 支持用户对生成的需求进行修改和确认

**FR-6**：数据分析执行
- 调用后端执行数据分析
- 支持统计分析和机器学习预测
- 处理执行过程中的错误和异常

### 5.4 结果展示

**FR-7**：图表可视化
- 将分析结果以图表形式展示
- 支持多种图表类型（柱状图、折线图、散点图、热力图等）
- 支持图表交互和导出

**FR-8**：分析报告
- 生成包含分析结果和图表的分析报告
- 支持报告的预览和导出

## 6. 非功能需求

**NFR-1**：性能要求
- 数据基本统计分析响应时间不超过 5 秒
- 复杂分析执行时间不超过 30 秒
- 页面响应时间不超过 2 秒

**NFR-2**：用户体验
- 界面简洁直观，操作流程清晰
- 提供详细的操作提示和帮助信息
- 支持实时的分析进度反馈

**NFR-3**：可扩展性
- 模块化设计，便于添加新的分析类型
- 支持不同类型的数据源
- 支持自定义分析选项

**NFR-4**：安全性
- 确保数据分析过程中的数据安全
- 防止恶意代码执行
- 数据传输加密

## 7. 约束

**技术约束**：
- 后端使用 Python 和 FastAPI
- 前端使用 Vue 3 和 Element Plus
- 数据库使用 PostgreSQL
- 大模型集成使用现有框架
- 参考 DataAgent-main 中的统计分析和机器学习预测代码

**业务约束**：
- 保持与现有系统的兼容性
- 遵循现有的用户权限管理
- 符合公司的安全规范

**依赖**：
- 依赖现有的数据源管理功能
- 依赖现有的大模型服务
- 依赖现有的用户认证系统

## 8. 假设

- 用户具备基本的数据分析知识
- 系统已经配置了可用的大模型
- 系统已经连接了至少一个数据源
- 服务器具备足够的资源执行数据分析
- DataAgent-main 中的统计分析和机器学习预测代码可以被参考和复用

## 9. 验收标准

### 9.1 数据基本统计分析验收

**AC-1**：数据基本统计信息
- **Given**：用户选择数据源和表名
- **When**：系统加载数据
- **Then**：系统显示行数、列数、各列数据类型和前五条数据
- **Verification**：`programmatic`

**AC-2**：数据质量分析
- **Given**：用户选择数据源和表名
- **When**：系统执行数据质量分析
- **Then**：系统显示缺失值、重复值等质量问题
- **Verification**：`programmatic`

### 9.2 智能分析建议验收

**AC-3**：常用统计分析选项
- **Given**：用户进入分析需求建议页面
- **When**：系统加载分析选项
- **Then**：系统显示常用的统计分析选项
- **Verification**：`human-judgment`

**AC-4**：列选择建议
- **Given**：用户选择分析类型
- **When**：系统提示列选择需求
- **Then**：系统明确提示需要选择一列或多列，并提供列名选择
- **Verification**：`human-judgment`

### 9.3 数据分析执行验收

**AC-5**：分析需求生成
- **Given**：用户选择分析类型和列名
- **When**：系统生成分析需求
- **Then**：系统显示生成的分析需求，支持用户修改
- **Verification**：`human-judgment`

**AC-6**：数据分析执行
- **Given**：用户确认分析需求
- **When**：系统执行数据分析
- **Then**：系统返回正确的分析结果
- **Verification**：`programmatic`

### 9.4 结果展示验收

**AC-7**：图表可视化
- **Given**：分析执行完成
- **When**：系统展示分析结果
- **Then**：系统以图表形式展示结果，支持多种图表类型
- **Verification**：`human-judgment`

**AC-8**：分析报告
- **Given**：分析执行完成
- **When**：系统生成分析报告
- **Then**：系统显示包含分析结果和图表的报告
- **Verification**：`human-judgment`
