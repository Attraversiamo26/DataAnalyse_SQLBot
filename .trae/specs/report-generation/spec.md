# 报告生成功能模块 - 产品需求文档

## Overview
- **Summary**: 完成报告生成功能页面，支持两种报告生成方式：1) 通过上传模板自动生成问题列表并调用分析工具；2) 从历史会话中选择问题进行汇总分析。
- **Purpose**: 提供自动化报告生成能力，让用户能够基于模板或历史会话快速生成专业的数据分析报告。
- **Target Users**: 数据分析人员、业务分析师、管理层

## Goals
- 实现模板上传功能，支持解析模板中的【重点关注】内容
- 实现大模型自动生成问题列表功能
- 实现调用智能问数/数据分析工具执行问题
- 实现大模型推理分析生成最终报告
- 实现从会话管理中选择历史会话生成报告
- 保持与会话管理模块的数据一致性

## Non-Goals (Out of Scope)
- 不支持报告模板的在线编辑功能
- 不支持报告的格式导出（如PDF、Word）
- 不支持报告的版本管理

## Background & Context
- 当前报告生成页面已具备基础框架，但核心功能未实现
- 需要调用现有的智能问数和数据分析工具API
- 需要与会话管理模块共享数据

## Functional Requirements
- **FR-1**: 用户可以上传报告模板（支持markdown格式）
- **FR-2**: 系统能够解析模板中的【重点关注】部分
- **FR-3**: 大模型能够根据模板内容生成问题列表
- **FR-4**: 系统能够调用智能问数或数据分析工具执行问题
- **FR-5**: 大模型能够基于问题和结果进行推理分析生成报告
- **FR-6**: 用户可以从历史会话中选择多个会话记录
- **FR-7**: 系统能够基于选中的会话记录生成综合报告
- **FR-8**: 报告生成过程支持进度展示
- **FR-9**: 生成的报告支持查看、编辑和下载

## Non-Functional Requirements
- **NFR-1**: 报告生成过程需要在30秒内完成（单模板或少量会话）
- **NFR-2**: 支持最大100个问题的批量处理
- **NFR-3**: 支持模板最大10MB
- **NFR-4**: 保证数据安全性，会话数据仅对当前用户可见

## Constraints
- **Technical**: 基于现有的Vue 3 + Element Plus框架，使用FastAPI后端
- **Business**: 需要与现有的智能问数、数据分析、会话管理模块集成
- **Dependencies**: 依赖大语言模型服务、数据库服务

## Assumptions
- 用户已经登录并有权限访问数据源
- 智能问数和数据分析工具API已正常运行
- 会话管理模块的数据结构稳定

## Acceptance Criteria

### AC-1: 模板上传与解析
- **Given**: 用户在报告生成页面
- **When**: 用户上传包含【重点关注】章节的markdown模板文件
- **Then**: 系统成功解析模板并提取【重点关注】内容
- **Verification**: `programmatic`

### AC-2: 问题列表生成
- **Given**: 用户上传了有效的模板
- **When**: 用户点击"生成问题列表"按钮
- **Then**: 系统调用大模型生成相关问题列表
- **Verification**: `programmatic`

### AC-3: 问题执行与报告生成
- **Given**: 已生成问题列表
- **When**: 用户点击"执行并生成报告"按钮
- **Then**: 系统依次执行问题并生成综合报告
- **Verification**: `programmatic`

### AC-4: 历史会话选择
- **Given**: 用户在报告生成页面
- **When**: 用户从会话列表中选择多个会话
- **Then**: 系统正确加载选中会话的问题和答案
- **Verification**: `programmatic`

### AC-5: 会话报告生成
- **Given**: 用户选中了多个历史会话
- **When**: 用户点击"生成综合报告"按钮
- **Then**: 系统基于选中会话生成综合分析报告
- **Verification**: `programmatic`

### AC-6: 报告展示与下载
- **Given**: 报告已生成
- **When**: 用户点击查看或下载按钮
- **Then**: 系统正确展示报告内容并支持下载
- **Verification**: `human-judgment`
