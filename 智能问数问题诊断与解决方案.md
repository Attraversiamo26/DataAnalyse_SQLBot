# 智能问数问题诊断与解决方案

## 问题描述

前端显示 "net::ERR_ABORTED" 错误，智能问数一直处于"思考中"状态，无法返回结果。

## 根本原因分析

通过分析后端日志，发现以下**数据库兼容性问题**：

### ❌ 问题 1：MySQL 不支持 PostgreSQL JSONB 语法

**错误信息**：
```
pymysql.err.ProgrammingError: (1064, "You have an error in your SQL syntax... near '@> jsonb_build_array(1))'")
```

**原因**：系统在 `terminology` 表查询中使用了 PostgreSQL 特有的 `jsonb_build_array` 和 `@>` 操作符，但你的配置是 MySQL。

**涉及文件**：
- [terminology.py](file:///Users/qiweideng/Desktop/DataAnalyse_SQLBot-main/backend/apps/terminology/curd/terminology.py)

### ❌ 问题 2：枚举类型属性缺失

**错误信息**：
```
AttributeError: type object 'CustomPromptTypeEnum' has no attribute 'GENERATE_SQL'
```

**原因**：`CustomPromptTypeEnum` 枚举类中缺少 `GENERATE_SQL` 属性定义。

**涉及文件**：
- [prompt_model.py](file:///Users/qiweideng/Desktop/DataAnalyse_SQLBot-main/backend/apps/system/models/prompt_model.py)
- [llm.py](file:///Users/qiweideng/Desktop/DataAnalyse_SQLBot-main/backend/apps/chat/task/llm.py#L666)

### ❌ 问题 3：SQLAlchemy 列表达式错误

**错误信息**：
```
sqlalchemy.exc.ArgumentError: Column expression expected, got <class 'sqlbot_xpack.permissions.models.ds_rules.DsRules'>
```

**原因**：`init_messages` 方法中的权限查询逻辑存在问题。

### 方案 2：修复 MySQL 兼容性（推荐）

#### 修复 1：terminology.py 中的 JSON 操作

找到 `select_terminology_by_word` 函数（约第 702 行），修改数据源筛选逻辑：

```python
# 原代码（PostgreSQL 语法）：
datasource_conditions.append(
    text("datasource_ids @> jsonb_build_array(:datasource)")
)

# 修改为 MySQL 兼容语法：
datasource_conditions.append(
    text(f"JSON_CONTAINS(datasource_ids, JSON_ARRAY({ds_id}))")
)
```

#### 修复 2：添加缺失的枚举属性

**修改文件**：[prompt_model.py](file:///Users/qiweideng/Desktop/DataAnalyse_SQLBot-main/backend/apps/system/models/prompt_model.py)

找到 `CustomPromptTypeEnum` 类定义，添加 `GENERATE_SQL = 1`：

```python
class CustomPromptTypeEnum(str, Enum):
    GENERATE_SQL = 1          # 添加这行
    ANALYSIS = 2
    PREDICT_DATA = 3
```

同时检查 `llm.py` 中的调用是否正确传递了枚举值：

```python
# 确保使用的是整数而不是字符串
CustomPromptTypeEnum.GENERATE_SQL
```

#### 修复 3：修复 init_messages 权限查询

**修改文件**：[llm.py](file:///Users/qiweideng/Desktop/DataAnalyse_SQLBot-main/backend/apps/chat/task/llm.py)

## 快速修复脚本

如果需要快速修复，可以运行以下命令：

```bash
# 停止后端
pkill -f "python main.py"

# 进入后端目录
cd backend

# 创建修复脚本
cat > fix_mysql_compat.py << 'EOF'
import re

# 修复 terminology.py
with open('apps/terminology/curd/terminology.py', 'r') as f:
    content = f.read()

# 替换 PostgreSQL JSON 操作符为 MySQL 语法
content = content.replace(
    'text("datasource_ids @> jsonb_build_array(:datasource)")',
    'text(f"JSON_CONTAINS(datasource_ids, JSON_ARRAY({ds_id}))")'
)

with open('apps/terminology/curd/terminology.py', 'w') as f:
    f.write(content)

print("terminology.py 修复完成")

# 修复 prompt_model.py
with open('apps/system/models/prompt_model.py', 'r') as f:
    content = f.read()

# 添加缺失的枚举值
if 'GENERATE_SQL = 1' not in content:
    content = content.replace(
        'class CustomPromptTypeEnum(str, Enum):',
        'class CustomPromptTypeEnum(str, Enum):\n    GENERATE_SQL = 1'
    )
    with open('apps/system/models/prompt_model.py', 'w') as f:
        f.write(content)
    print("prompt_model.py 修复完成")
else:
    print("prompt_model.py 无需修复")

print("所有修复完成！")
EOF

# 运行修复脚本
python fix_mysql_compat.py

# 删除修复脚本
rm fix_mysql_compat.py

# 重启后端
python main.py
```
