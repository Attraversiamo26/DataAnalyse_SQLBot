# 线路五环节时限对标分析

## 功能描述

此 skill 用于分析寄递线路五环节时限对标分析数据，读取 Excel 数据后自动更新 PPT 模板中的图表和文字内容，并生成 PDF 格式的分析结果摘要。

**核心功能**：
1. 读取上级目录 `/data/寄递时限对标分析指标大全.xlsx` 文件中的 `线路表` sheet
2. 根据输入的线路名称，匹配线路名字段，筛选数据
3. 自动更新 PPT 模板中的图表和文字内容
4. 生成 PDF 格式的分析结果摘要

**输出格式**：
- 自动生成 PPT 分析报告（保持模板格式）：`线路五环节时限对标分析结果_[线路名称].pptx`
- 自动生成分析结果摘要 PDF：`线路五环节时限对标分析结果摘要_[线路名称].pdf`
- 分析完成后提示"相关分析已经保存为 PPT 及报告"

## 使用场景

当用户需要：
- 基于线路接口数据生成标准化的五环节时限对标分析
- 开展线路五环节时限对标分析
- 开展线路时限对标分析



**触发示例**：
- "开展北京到上海线路五环节时限对标分析"
- "生成广州 - 深圳线路的五环节分析报告"

## 分析逻辑

1. **五环节时限**：收寄、出口、中转、进口、投递
2. **关键环节识别**：计算各环节与竞品的差值绝对值，找出差值最大的环节
3. **PPT 文字替换**：将模板中的"进口段"自动替换为差值最大的环节名称（如"中转段"），**仅替换 PPT 上半部分文本框一次，下方表格保持不变**
4. **差距值替换**：将模板中"进口段"的差距值（4.7 小时）替换为关键环节的实际差距值（如中转段差距 5.7 小时）
5. **数据更新**：使用精确的 XML 标签匹配更新所有时限数据和差异值，确保只替换第一个匹配项，保持表格数据不变

## 实现细节

**PPT 数据替换策略**：
- 使用正则表达式 `/(<a:t>)数字(<\/a:t>)/` 精确匹配 XML 中的文本标签
- 按顺序替换各环节数据，避免重复替换或错误替换
- 只替换每个数字的第一个匹配项，确保表格中的原始数据保持不变
- 替换顺序：总时限 → 总差异值 → 环节名称 → 关键环节差距值 → 特快各环节 → 竞品各环节

**模板原始数据与替换映射**：

总时限相关：
- `30.1` → 我方总时限
- `21.4` → 竞品总时限
- `8.7` → 总时限差距绝对值
- `4.7` → 关键环节差距值（根据差值最大的环节动态替换）

特快数据替换映射：
- `2.2` → 收寄时限
- `8.6` → 出口时限
- `6.9` → 中转时限
- `7.9` → 进口时限
- `4.5` → 投递时限

竞品数据替换映射：
- `2.4` → 收寄时限
- `6.6` → 出口时限
- `6.5` → 中转时限
- `3.2` → 进口时限
- `2.6` → 投递时限

## 术语说明

- **中转段时限** = **中转时限**（同一概念的不同表述）
- **进口段时限** = **进口时限**（同一概念的不同表述）
- PPT 模板中使用"段"字表述（如中转段、进口段）
- 数据表格中使用简洁表述（如中转时限、进口时限）

## 注意事项

- 数据文件格式需与模板期望的格式一致
- 模板文件必须存在且格式正确
- 需要安装 Node.js 依赖：`npm install xlsx jszip pdfkit`
- 生成的 PPT 完全保持模板的格式、样式和布局
- 输出文件保存在 `输出结果` 文件夹中
- 分析结果摘要以 PDF 格式保存（使用英文避免字体问题）
