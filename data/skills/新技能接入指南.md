# 技能管理器 - 新技能接入指南

## 一、概述

技能管理器（Skill Manager）是数据分析平台的核心模块，负责技能的加载、匹配、执行和链式触发。每个技能是一个独立目录，包含脚本代码、描述文件和工作流定义。

### 技能三层架构

| 层级 | layer 值 | 说明 | 示例 |
|------|----------|------|------|
| 全景分析层 | `overview` | 整体线路表现的全景分析 | 五环节时限对标分析 |
| 环节分析层 | `stage` | 针对特定环节的下钻分析 | 出口/进口/投递/中转/收寄环节分析 |
| 机构下钻层 | `institution` | 针对特定机构的深度分析 | 出口/收寄机构48小时分布分析 |

### 技能执行流程

```
用户输入 → 意图识别 → 技能匹配 → 参数提取 → 工作流步骤执行 → 结构化输出 → 链式触发（可选）
```

## 二、技能目录结构

每个技能必须位于 `data/skills/` 目录下的独立子目录中：

```
data/skills/
└── your-skill-name/              # 技能目录（使用小写英文和连字符）
    ├── SKILL.md                  # 技能描述文件（必需）
    ├── workflow.yaml             # 工作流定义文件（必需）
    ├── analyze.js                # 主入口脚本（Node.js）
    │   或 analyze.py             # 主入口脚本（Python）
    ├── check_sheets.js           # 辅助脚本（可选）
    ├── scripts/                  # 子脚本目录（可选）
    │   ├── chart_48h.py          # 图表生成脚本
    │   └── generate_report.py    # 报告生成脚本
    ├── 输入数据/                  # 本地数据文件（可选，优先使用共享数据）
    │   └── 接口文档-0401(1).xlsx
    ├── 输出结果/                  # 输出目录（可选）
    ├── node_modules/             # Node.js 依赖（如有）
    ├── package.json              # Node.js 包配置（如有）
    └── requirements.txt          # Python 依赖（如有）
```

## 三、编写 SKILL.md

`SKILL.md` 是技能的描述文件，系统通过它了解技能的功能、触发条件和参数。

### 完整模板

```markdown
---
id: your-skill-short-id
name: 技能中文名称
description: 技能的详细描述，说明该技能做什么、输出什么
layer: overview|stage|institution
inputs:
  - name: route_name
    type: string
    required: true
    description: 线路名称，如 北京 - 上海
  - name: top_n
    type: number
    required: false
    default: 5
    description: 取前N个机构
outputs:
  - name: ppt_report
    type: file
    description: PPT分析报告
  - name: key_problem_stage
    type: string
    description: 问题最严重的环节
---

# 技能中文名称

## 功能说明

详细描述该技能的功能、分析逻辑和输出内容。

## 触发示例

- "开展北京到上海线路五环节时限对标分析"
- "分析出口环节的慢机构"
- "看看滨州市大同市线路的出口机构"

## 参数说明

*第一个参数*：线路名称（必填），格式如"滨州市 - 大同市"
*第二个参数*：取前N名（可选，默认5）

## 输出文件

- PPT报告：输出结果/分析结果.pptx
- PDF摘要：输出结果/分析结果摘要.pdf

## 联动触发

当分析完成后，可联动触发下游技能：
- 出口环节 → 出口机构下钻分析
- 收寄环节 → 收寄机构下钻分析
```

### YAML Frontmatter 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 是 | 技能短ID，如 `export-link`、`pickup-institution` |
| `name` | string | 是 | 技能中文名称 |
| `description` | string | 是 | 技能描述 |
| `layer` | string | 是 | 技能层级：`overview`/`stage`/`institution` |
| `inputs` | array | 否 | 输入参数定义列表 |
| `outputs` | array | 否 | 输出定义列表 |

### inputs 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 参数名称（英文，如 `route_name`） |
| `type` | string | 是 | 参数类型：`string`/`number`/`boolean` |
| `required` | boolean | 是 | 是否必填 |
| `default` | any | 否 | 默认值 |
| `description` | string | 是 | 参数描述 |

### 触发示例的重要性

触发示例是意图识别的关键依据。系统会将用户输入与触发示例进行匹配，匹配度越高，该技能被选中的概率越大。建议：

- 提供 3-5 个不同表述方式的触发示例
- 覆盖不同的用户表达习惯
- 包含关键词（环节名称、机构、线路等）

## 四、编写 workflow.yaml

`workflow.yaml` 定义了技能的内部执行步骤，包括脚本调用顺序、参数传递、依赖关系等。

### 完整模板

```yaml
id: your-skill-short-id
name: 技能中文名称
description: 技能描述
data_file: "../data/寄递时限对标分析指标大全.xlsx"
env_requirements:
  - "node >= 14"
  - "npm install xlsx pdfkit"
  - "python >= 3.8"
  - "pip install pandas python-docx docx2pdf matplotlib numpy openpyxl"

steps:
  - id: step_1_validate_data
    name: 验证数据文件
    description: 检查Excel数据文件是否存在
    script: "check_sheets.js"
    language: nodejs
    args_template: []
    timeout: 30
    optional: true

  - id: step_2_read_data
    name: 读取分析数据
    description: 从数据文件中读取指定数据
    script: "analyze.js"
    language: nodejs
    args_template:
      - "${params.route_name}"
      - "${params.top_n}"
    depends_on: ["step_1_validate_data"]
    timeout: 60
    outputs:
      - name: ppt_report
        type: file
        value: "输出结果/分析结果_${params.route_name}.pptx"
      - name: key_problem_stage
        type: string
        value: "${step_2_read_data.output.key_problem_stage}"

  - id: step_3_generate_report
    name: 生成详细报告
    description: 生成Word/PDF格式的详细分析报告
    script: "scripts/generate_report.py"
    language: python
    args_template: []
    depends_on: ["step_2_read_data"]
    condition: "${step_2_read_data.success} == True"
    timeout: 120
    optional: true
    outputs:
      - name: analysis_report
        type: file
        value: "scripts/分析报告_*.docx"
      - name: pdf_report
        type: file
        value: "scripts/分析报告_*.pdf"
```

### 顶层字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 是 | 技能短ID，与 SKILL.md 中的 id 一致 |
| `name` | string | 是 | 技能名称 |
| `description` | string | 否 | 技能描述 |
| `data_file` | string | 否 | 数据文件路径（相对于技能目录） |
| `env_requirements` | array | 否 | 环境依赖列表 |
| `steps` | array | 是 | 工作流步骤列表 |

### steps 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | string | 是 | - | 步骤ID，唯一标识 |
| `name` | string | 否 | - | 步骤名称 |
| `description` | string | 否 | - | 步骤描述 |
| `script` | string | 是 | - | 脚本路径（相对于技能目录） |
| `language` | string | 是 | - | 脚本语言：`nodejs`/`python` |
| `args_template` | array | 否 | `[]` | 参数模板列表 |
| `depends_on` | array | 否 | `[]` | 依赖的前置步骤ID列表 |
| `condition` | string | 否 | - | 执行条件表达式 |
| `timeout` | int | 否 | `120` | 步骤超时时间（秒） |
| `optional` | boolean | 否 | `false` | 是否可选（失败不阻断后续步骤） |
| `outputs` | array | 否 | `[]` | 步骤输出定义 |

### 参数模板语法

`args_template` 支持以下变量替换：

| 语法 | 说明 | 示例 |
|------|------|------|
| `${params.xxx}` | 用户传入的参数 | `${params.route_name}` → `滨州市 - 大同市` |
| `${steps.prev_id.output.xxx}` | 前置步骤的输出 | `${step_1.key_problem_stage}` → `进口` |

### 条件表达式语法

`condition` 支持以下变量替换和 Python 表达式：

```yaml
condition: "${step_1.success} == True"
condition: "${step_1.output.key_problem_stage} == '进口'"
condition: "${params.top_n} > 5"
```

### outputs 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 输出名称 |
| `type` | string | 是 | 输出类型：`file`/`string`/`array`/`object`/`number` |
| `value` | string | 否 | 输出值表达式 |
| `description` | string | 否 | 输出描述 |

### 5.1 添加短ID映射

编辑 `backend/apps/skill_manager/skill_manager.py`，在 `SHORT_ID_MAP` 中添加：

```python
SHORT_ID_MAP = {

}
```

**示例**：如果技能目录名为 `export-link-drilldown-analysis`，短ID为 `export-link`：

```python

```

### 5.2 添加数据Sheet映射（可选）

如果技能使用了特定的 Excel Sheet，在 `DATA_SHEETS_MAP` 中添加：

```python
DATA_SHEETS_MAP = {
    # ... 已有映射 ...
    'your-skill-directory-name': ["Sheet名称1", "Sheet名称2"],
}
```

### 5.3 添加结构化输出定义（可选）

如果技能需要输出结构化数据（用于链式触发参数传递），在 `SKILL_OUTPUTS_MAP` 中添加：

```python
SKILL_OUTPUTS_MAP = {
    # ... 已有映射 ...
    'your-skill-directory-name': [
        SkillOutputDef(name="slowest_institution", type="string",
                       value="${step_4_get_top_n.slow_institutions[0].机构名称}"),
        SkillOutputDef(name="key_problem_stage", type="string",
                       value="${step_4_identify_problem.problem_analysis.max_gap_stage}"),
    ],
}
```

### 5.4 添加链式触发（可选）

如果技能执行成功后需要触发下游技能，在 `CHAIN_TRIGGERS_MAP` 中添加：

```python
CHAIN_TRIGGERS_MAP = {
    # ... 已有映射 ...
    'your-skill-directory-name': [
        SkillChainTrigger(
            skill_id='downstream-skill-directory-name',
            trigger_type='on_user_confirm',  # 或 'on_success'
            params_mapping={
                'institution_name': '${outputs.slowest_institution}',
                'route_name': '${params.route_name}'
            }
        )
    ],
}
```

**触发类型说明**：

| 类型 | 说明 |
|------|------|
| `on_success` | 主技能执行成功后自动触发 |
| `on_user_confirm` | 主技能执行成功后，等待用户确认再触发 |

**参数映射语法**：

| 语法 | 说明 |
|------|------|
| `${outputs.xxx}` | 引用主技能的结构化输出 |
| `${params.xxx}` | 引用用户传入的参数 |

### 5.5 添加专有关键词（推荐）

为了提高意图识别的准确率，在 `match_skills()` 方法的 `stage_specific_keywords` 中添加：

```python
stage_specific_keywords = {
    # ... 已有关键词 ...
    'your-skill-directory-name': ["关键词1", "关键词2", "关键词3"],
}
```

这些关键词在用户查询中出现时，会显著提高该技能的匹配置信度（+0.3）。

### 6.3 PDF 生成规范

Word 转 PDF 需要安装 LibreOffice。脚本中的 `convert_to_pdf` 函数应支持多种转换方式：

```python
def convert_to_pdf(docx_path):
    pdf_path = os.path.splitext(docx_path)[0] + '.pdf'

    # 方式1: docx2pdf（Windows/macOS）
    try:
        from docx2pdf import convert
        convert(docx_path, pdf_path)
        return pdf_path
    except Exception:
        pass

    # 方式2: win32com（Windows）
    try:
        import win32com.client
        word = win32com.client.Dispatch('Word.Application')
        word.Visible = False
        doc = word.Documents.Open(docx_path)
        doc.SaveAs(pdf_path, FileFormat=17)
        doc.Close()
        word.Quit()
        return pdf_path
    except Exception:
        pass

    # 方式3: LibreOffice（macOS/Linux）
    import subprocess
    try:
        subprocess.run(
            ['libreoffice', '--headless', '--convert-to', 'pdf',
             '--outdir', os.path.dirname(pdf_path), docx_path],
            capture_output=True, text=True, timeout=60
        )
        if os.path.exists(pdf_path):
            return pdf_path
    except Exception:
        pass

    print('PDF 转换失败，仅生成 Word 文档')
    return None
```

## 九、常见问题

### Q1: 技能加载后不显示？

检查 `SKILL.md` 文件是否存在且格式正确。YAML frontmatter 必须用 `---` 包裹。

### Q2: 意图识别匹配不到？

1. 检查 `SKILL.md` 中的触发示例是否包含用户可能使用的关键词
2. 在 `stage_specific_keywords` 中添加专有关键词
3. 触发示例越具体，匹配越准确

### Q3: 脚本执行失败？

1. 检查脚本路径是否正确（相对于技能目录）
2. 检查 `workflow.yaml` 中的 `language` 是否与脚本扩展名一致
3. 检查数据文件路径是否使用了共享路径
4. 检查环境依赖是否已安装

### Q4: PDF 没有生成？

需要安装 LibreOffice：
```bash
# macOS
brew install --cask libreoffice

# Ubuntu/Debian
sudo apt install libreoffice

# Windows
# 下载安装 https://www.libreoffice.org/download/
```

### Q5: 如何热重载技能？

调用 API 端点：
```bash
curl -X POST http://localhost:8000/skill-manager/skills/reload
```

### Q6: 如何调试工作流步骤？

查看每个步骤的标准输出和错误输出：
```python
result = await sm.execute_skill_steps('your-skill', params={})
print(result.output)   # 所有步骤的标准输出
print(result.error)    # 所有步骤的错误输出
```
