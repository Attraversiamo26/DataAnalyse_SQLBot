# 收寄环节下钻分析

## 功能描述

此 skill 用于分析指定线路的收寄环节下钻数据，读取 Excel 数据后找出收寄时间最长的 N 个机构，并生成分析总结 PDF。

**输出格式**：
- 直接在控制台输出分析结果
- 分析完成后显示详细分析总结
- **询问用户**：询问是否要对收寄时间最长的机构开展下钻分析

**联动功能**：
- 如果用户选择需要，会自动调用 `export-institution-drilldown-analysis` Skill，对该机构开展出口环节下钻分析

## 使用场景

当用户需要：
- 分析某线路收寄环节的具体机构表现
- 找出收寄时间最长的机构
- 了解收寄环节慢的原因

**触发示例**：
- "分析北京到上海线路的收寄环节"
- "北京-上海线路收寄环节下钻分析"
- "看看广州-深圳线路收寄哪些机构最慢"
- "滨州市到大同市的线路收寄环节"

## 文件位置

- **数据文件**：`../data/寄递时限对标分析指标大全.xlsx`
- **输出文件**：无（直接输出到控制台）

## 执行方式

```bash
cd c:\Users\TJY\Desktop\对标 skill\.trae\skills\pickup-stage-drilldown-analysis
node analyze.js "北京-上海"
```

**参数说明**：
- 第一个参数：线路名称（必填），如 `"北京-上海"`
- 第二个参数：机构数量 N（可选，默认 5），如 `5`

## 数据结构

**收寄机构表** sheet 包含以下关键列：
- [0] 线路名（如：北京市-上海市）
- [1] 机构名称
- [2] 收寄时限（小时）
- [3] 特快收寄时限（小时）
- [4] 竞品收寄时限（小时）
- [5] 差值（小时）

## 分析逻辑

1. **数据筛选**：根据线路名称筛选数据
2. **排序**：按收寄时间从长到短排序
3. **Top N 提取**：取出收寄时间最长的 N 个机构（默认 5 个）
4. **对比分析**：分析这些机构与竞品的差距
5. **原因总结**：总结机构收寄慢的主要原因

## 实现细节

**分析维度**：
- 机构收寄时限绝对值
- 与竞品收寄时限的差值
- 差值为正表示落后，为负表示领先
- 重点关注差值较大的机构

**PDF 内容结构**：
1. 线路基本信息
2. Top N 机构列表（按收寄时间排序）
3. 各机构详细数据
4. 分析总结与建议

## 术语说明

- **收寄时限**：从客户交寄到邮件进入处理环节的时间
- **差值**：特快收寄时限 - 竞品收寄时限
- **正差值**：表示比特快慢，需要改进
- **负差值**：表示比特快快，保持优势

## 注意事项

- 数据文件格式需与模板期望的格式一致
- 需要安装 Node.js 依赖：`npm install xlsx`
- 分析结果直接输出到控制台
- 默认分析收寄时间最长的 5 个机构
